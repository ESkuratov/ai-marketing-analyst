# Сквозная воронка: маппинг источников и бизнес-решения

## Таблица соответствия этапов

| № | Этап воронки | Google Sheets | AMO CRM | YClients | Бизнес решение |
|---|-------------|---------------|---------|----------|----------------|
| 1 | **Новая заявка** | `client_phone`, `source_name`, `utm_source`, `utm_campaign`, `created_at` | `client_phone`, `stage_id IN (74087434, 74087430)`, `created_at` | — | Для всех клиентов в AMO со статусом "Новая заявка" / "Неразобранное" выполнить матчинг по `client_phone` с Google Sheets. Если найден — записать `utm_source` / `utm_campaign` в лид (обогащение). Если не найден — уведомить в ТГ (лид не из рекламы) |
| 2 | **Переговоры** | — | `client_phone`, `stage_id IN (84866070, 74087586)`, `created_at`, `updated_at` | — | Замерить длительность переговоров: `updated_at - created_at`. Если > N дней без смены статуса — уведомить в ТГ «клиент завис в переговорах» |
| 3 | **Не взяли трубку** | — | `client_phone`, `stage_id IN (74087582, 84866066)`, `updated_at` | — | Отслеживать количество попыток и время с последнего контакта. Если > N дней без смены статуса — уведомить «клиент потерян» |
| 4 | **Назначен визит** | — | `client_phone`, `stage_id = 74087590`, `created_at` | `client_phone`, `status IN ('confirmed', 'waiting')`, `date`, `time` | Для каждого клиента в AMO со статусом "Назначен визит" проверить наличие записи в YC со статусом `confirmed` / `waiting`. Матчинг по `client_phone`. Если записи нет — уведомить в ТГ «визит назначен в AMO, но не создан в YC» |
| 5 | **Не пришла** | — | `client_phone`, `stage_id = 74087594` | `client_phone`, `status IN ('visited', 'not_visited')`, `date` | Для клиентов в AMO со статусом "Не пришла" проверить статус в YC. Если YC показывает `visited` — расхождение, клиент фактически был. Уведомить в ТГ. Если оба показывают `not_visited` — OK |
| 6 | **Дошла до 1 записи** | — | `client_phone` (идентификация клиента) | `client_phone`, `is_first_visit = true`, `status = visited`, `date`, `service_name`, `master_name`, `sum` | Положительный этап. Отслеживается по YC: первый визит клиента. Отметить в `client_profiles` (`funnel_stage = completed`). Уведомить в ТГ менеджера для upsell (предложить абонемент) |
| 7 | **Не купила** | — | `client_phone`, `stage_id = 74087598` | `client_phone`, `status`, `sum`, visit history | Для клиентов в AMO со статусом "Не купила" проверить историю посещений в YC. Если YC показывает `visits > 0` — клиент был на сеансе, но не купил → уведомить в ТГ «был визит, но нет продажи». Если `visits = 0` — клиент не дошёл, ожидаемо |
| 8 | **Внесла залог** | — | `client_phone`, `stage_id = 84237074`, `price` | `client_phone`, `sum`, `status`, `service_name` | Для клиентов в AMO со статусом "внесла залог" проверить сумму в YC. Матчинг по `client_phone`. Если `YC.sum < AMO.price` или отсутствует — расхождение, уведомить в ТГ. Сравнить: ожидаемый залог vs фактическая оплата |
| 9 | **Успешно реализовано** | — | `client_phone`, `stage_id = 142`, `price` | `client_phone`, `sum`, `status = visited`, visit history | Для клиентов в AMO со статусом "Успешно реализовано" проверить общую сумму продаж в YC. Если `YC.sum < AMO.price` — не хватает данных о продаже. Уведомить в ТГ с суммой и информацией о клиенте |
| 10 | **Не продлила** | — | `client_phone`, `stage_id = 83303646`, `closed_at` | `client_phone`, `date` (последний визит), `service_name`, `master_name`, `comment`, visit history | Для клиентов в AMO со статусом "Не продлила" проверить, когда был последний визит в YC. Если `NOW() - last_visit > N дней` — подтверждение. Собрать историю услуг (`service_name`, `master_name`) и комментарии из старых записей (пожелания, проблемные зоны, предпочтения). Уведомить в ТГ: количество визитов, дата последнего, на какие услуги ходила, ключевые комментарии → предложить персонализированную реактивацию |
| 11 | **Продлила** | — | `client_phone`, `stage_id = 84237070` (Ходит разово) | `client_phone`, `is_repeat = true`, visit count, `sum` | Положительный этап. Если в YC есть регулярные посещения (`is_repeat`), а в AMO статус не обновлён — обновить. Уведомить в ТГ: предложить долгосрочный абонемент / программу лояльности |
| 12 | **Закрыто и не реализовано** | — | `client_phone`, `stage_id = 143`, `closed_at`, `price`, `utm_source`, `utm_campaign` | — | Аналитика причин: по UTM-каналам, по сумме, по длительности ведения. Если процент закрытых без продажи > порога — уведомить в ТГ |

## Условные обозначения

| Символ | Значение |
|--------|----------|
| — | Источник не участвует на этом этапе |
| GS | Google Sheets — лиды из рекламных каналов |
| AMO | AMO CRM — ведение клиента по воронке |
| YC | YClients — записи и посещения |

## Ключевые параметры матчинга

| Матчинг | Параметр | Где определён |
|---------|----------|---------------|
| AMO ↔ YC | `client_phone` (нормализованный: `7XXXXXXXXXX`) | S2 enrichment (`s2_normalize.sql`) |
| AMO ↔ GS | `client_phone` (нормализованный) | S3 reconcile (`s3_reconcile.sql`) |
| AMO → client_profile | `client_phone` | S2b (`s2b_client_profiles.sql`) |
| YC → client_profile | `client_phone` | S2b (`s2b_client_profiles.sql`) |
| GS → client_profile | `client_phone` | S2b (`s2b_client_profiles.sql`) |

## YClients статусы

| Статус API | Маппинг в pipeline | Описание |
|-----------|-------------------|----------|
| `visited` | `completed` | Факт посещения |
| `not_visited` | `no_show` | Не пришла на сеанс |
| `canceled` | `canceled` | Отмена записи |
| `confirmed` | — | Подтвердила запись (предзапись) |
| `waiting` | — | Ожидание (зелёный, предзапись) |
